Run the Browserbase VM e2e with `source ~/misc/env.txt && pnpm --dir packages/core exec vitest run tests/browserbase-e2e.test.ts`. The test aliases `BROWSER_BASE_API_KEY` and `BROWSER_BASE_PROJECT_ID` from `~/misc/env.txt` to the `BROWSERBASE_*` names the CLI expects, projects the Browserbase CLI packages into the VM through `moduleAccessCwd`, creates/releases the remote session from inside the VM with a tiny guest `fetch()` helper, and drives navigation plus screenshot capture through `browse --ws <connectUrl>` while direct guest egress to unrelated domains stays denied.
