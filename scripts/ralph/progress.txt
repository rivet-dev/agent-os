# Ralph Progress Log
## Codebase Patterns
- Security tests must use real Node.js execution and attempt actual forbidden operations, not just verify argument construction with fake binaries.
- Sidecar test assertions must check response payload contents, not just match on variant shapes with wildcard discards.
- Import-cache tests should use `NodeImportCache::new_in()` with unique temp directories to avoid parallel test interference.
- Guest Node sync-RPC bootstrap FDs should be left for process teardown; closing inherited raw FDs with `fs.closeSync()` emits host warnings on newer Node builds.
- Full crate test suites (`cargo test -p <crate>`) must pass, not just focused individual test cases.
- After package renames or workspace dependency changes, validate from the repo root with `pnpm install --frozen-lockfile`; package-local checks can miss broken `workspace:*` resolution.
- SSRF filters must cover all RFC special-purpose ranges: 0.0.0.0/8, 10/8, 127/8, 169.254/16, 172.16/12, 192.168/16, 224/4, 255.255.255.255, plus IPv6 equivalents.
- S3-compatible loopback endpoints for local Minio or mock servers must opt in explicitly with `allowLoopbackEndpoint`; production defaults should keep loopback/private endpoint SSRF checks enabled.
- Native-sidecar Node launches must share the same host shadow root between `createVm` metadata `cwd` and `NativeSidecarKernelProxy.shadowRoot`; otherwise sidecar execute rejects the Node cwd as escaping the VM sandbox root.
- External mounts must be excluded from root filesystem quotas both when measuring usage and when enforcing per-path write/mkdir/truncate limits, or host-dir mounts will consume guest quota.
- Node import prewarm should use `node:` builtin specifiers instead of `agent-os:` synthetic URLs to stay compatible with newer Node ESM warmup behavior.
- Native-sidecar WASM executions read and write against the host `shadowRoot`, so the shadow root must include the standard POSIX bootstrap directories up front; missing `/tmp` breaks shell redirection and file-based command tests even when the guest root filesystem contains `/tmp`.
- If the WASM runner virtualizes `path_open`, it also needs matching virtual-FD handlers for follow-on syscalls such as `fd_read`, `fd_write`, `fd_seek`, `fd_tell`, `fd_pread`, `fd_pwrite`, and `fd_filestat_get`; shell redirection and coreutils file commands assume the full open-file-description lifecycle works after `dup`/`dup2`.
