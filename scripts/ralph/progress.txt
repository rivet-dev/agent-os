## Codebase Patterns
- Treat `scripts/ralph/archive/` as historical-only context; use the active `scripts/ralph/prd.json` test policy and per-story acceptance commands to decide what is green.
- The renamed core workspace package is `@rivet-dev/agent-os-core`; use that name for both package dependencies and TypeScript subpath imports like `@rivet-dev/agent-os-core/test/*` and `@rivet-dev/agent-os-core/internal/runtime-compat`.
- Root install reproducibility is best verified with the three-part probe `pnpm install --frozen-lockfile`, `pnpm check-types`, and a package-local `pnpm --dir <pkg> exec <bin> --version` check to prove the root install materialized script binaries for workspace packages.
- Node builtin compatibility regressions are best covered in `crates/sidecar/tests/builtin_conformance.rs`, which runs the same probe under host Node and guest V8 and compares the JSON output directly.
- The shared-runtime `node:stream` path exercised by sidecar builtin conformance comes from the inline mini-stream module in `crates/execution/src/javascript.rs`; fixing `require("stream")` parity there is separate from the stdlib-backed `v8-bridge.source.js` path.
- In the WASM host-process bridge, synthetic pipes must initialize both `producers` and `consumers`, and consumer registration must flush any chunks buffered before the child attaches; shell builtins can write into a pipe before a spawned reader like `wc` registers stdin.
- WASM tests that consume `poll_event_blocking()` directly need to match `WasmExecutionEvent::SyncRpcRequest(_)` explicitly unless they are intentionally asserting the control-plane payload; the runtime now surfaces sync RPC traffic in the same stream as stdout/stderr/signal/exit events.
- Package-provided agent descriptors registered via `processSoftware()` override the hardcoded `AGENT_CONFIGS` entries, so launch-critical defaults like Claude shell/env flags need to stay mirrored across both config surfaces.
- Claude’s patched SDK/CLI artifacts are located through `registry/agent/claude/dist/claude-*-patched.json`; regression tests should validate those manifests and fallback resolution with a temp package dir instead of mutating the real built artifacts.
- `packages/core/tests/claude-session.test.ts` is the end-to-end Claude truth suite; it exercises the real SDK path with llmock, PATH-backed `xu`, nested `node` child processes, metadata, modes, and lifecycle APIs.
- The sidecar TLS reader and writer share one `rustls::StreamOwned` mutex, so timed-out TLS read loops must yield briefly or guest HTTPS request writes can be starved and undici/fetch will see EOF before the request leaves the VM.
