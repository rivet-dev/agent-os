- The renamed core workspace package is `@rivet-dev/agent-os-core`; use that name for both package dependencies and TypeScript subpath imports like `@rivet-dev/agent-os-core/test/*` and `@rivet-dev/agent-os-core/internal/runtime-compat`.
- Node builtin compatibility regressions are best covered in `crates/sidecar/tests/builtin_conformance.rs`, which runs the same probe under host Node and guest V8 and compares the JSON output directly.
- The shared-runtime `node:stream` path exercised by sidecar builtin conformance comes from the inline mini-stream module in `crates/execution/src/javascript.rs`; fixing `require("stream")` parity there is separate from the stdlib-backed `v8-bridge.source.js` path.
- In the WASM host-process bridge, synthetic pipes must initialize both `producers` and `consumers`, and consumer registration must flush any chunks buffered before the child attaches; shell builtins can write into a pipe before a spawned reader like `wc` registers stdin.
- Package-provided agent descriptors registered via `processSoftware()` override the hardcoded `AGENT_CONFIGS` entries, so launch-critical defaults like Claude shell/env flags need to stay mirrored across both config surfaces.

- Files changed: `crates/execution/src/node_import_cache.rs`, `crates/execution/CLAUDE.md`, `scripts/ralph/prd.json`, `scripts/ralph/progress.txt`.
- **Learnings for future iterations:**
  - Shell builtins can write into a synthetic pipe before a spawned child has registered its stdin consumer, so buffered chunks must flush on consumer registration rather than only on later writes.
  - Producer teardown must consider the synthetic write-handle count too; once both producer registrations and write handles reach zero, attached child stdin should be closed immediately or readers like `wc` will block forever waiting for EOF.
  - Focused validation for this path is `cargo test -p agent-os-sidecar -- --test-threads=1 wasm`, `pnpm --dir packages/core exec vitest run tests/process-management.test.ts`, and the `tests/wasm-commands.test.ts` command-substitution coverage.
